## 📚 功能一：Method 库管理

### 查看预置方法

1. 系统启动后，默认显示 Method 库页面
2. 您会看到 4 个预置的交易方法：
   - 突破交易法（趋势跟踪）
   - 回调入场法（趋势跟踪）
   - 区间震荡法（震荡交易）
   - Pin Bar 反转（形态交易）

3. 每个方法卡片显示：
   - 方法名称
   - 分类标签（不同颜色）
   - 简要描述
   - 详细交易规则
   - 胜率统计
   - 更新日期

### 添加新方法

1. 点击右上角 **"添加新方法"** 按钮
2. 填写表单：
   ```
   方法名称：趋势线突破法
   分类：突破交易
   简要描述：利用趋势线突破捕捉趋势启动点
   交易规则：
   1. 绘制有效趋势线（至少3个触点）
   2. 等待价格突破趋势线
   3. 突破后回踩确认
   4. 在回踩位置入场
   5. 止损设在趋势线下方
   ```
3. 点击 **"确定"** 保存
4. 新方法卡片出现在列表中

### 编辑方法

1. 找到任意方法卡片
2. 点击右上角的 **编辑图标** ✏️
3. 修改信息（如更新规则）
4. 点击 **"确定"** 保存
5. 卡片内容立即更新

### 删除方法

1. 点击方法卡片右上角的 **删除图标** 🗑️
2. 确认删除
3. 方法从列表中移除

## 📊 功能二：交易复盘

### 查看交易记录

1. 点击顶部导航栏的 **"交易复盘"**
2. 看到表格展示的 8 条预置交易记录
3. 表格列包括：
   - 货币对
   - 方向（做多/做空）
   - 入场价/出场价
   - 盈亏（带颜色标识）
   - 结果（盈利/亏损/保本）
   - 使用方法
   - 入场时间
   - 标签
   - 操作按钮

### 添加新交易记录

1. 点击右上角 **"添加交易记录"** 按钮
2. 填写详细信息：

   **基本信息：**
   ```
   货币对：EUR/USD
   交易方向：做多
   入场价格：1.0850
   出场价格：1.0920
   手数：0.1
   ```

   **时间信息：**
   ```
   入场时间：2024-12-13 09:30:00
   出场时间：2024-12-13 15:45:00
   ```

   **方法和结果：**
   ```
   使用方法：突破交易法
   交易结果：盈利
   风险回报比：2.5
   ```

   **附加信息：**
   ```
   标签：趋势,突破,欧元
   交易笔记：欧元突破1.0850阻力位，回踩确认后入场，
            目标位1.0920达成。执行完美，严格遵守规则。
   ```

3. 点击 **"确定"** 保存
4. 新记录出现在表格顶部

### 编辑交易记录

1. 找到任意交易记录
2. 点击 **"编辑"** 按钮
3. 修改信息（如补充交易笔记）
4. 保存更改

### 删除交易记录

1. 点击 **"删除"** 按钮
2. 确认删除
3. 记录从表格中移除

### 筛选功能（未来版本）

- 按货币对筛选
- 按方法筛选
- 按结果筛选
- 按日期范围筛选

## 📈 功能三：我的统计

### 查看货币对统计

1. 左下方表格显示 **"货币对统计"**
2. 列出每个货币对的：
   - 交易次数
   - 盈利次数
   - 胜率
   - 总盈亏
3. 可以看出哪些货币对表现最好

### 查看方法统计

1. 右下方表格显示 **"方法统计"**
2. 列出每个方法的：
   - 使用次数
   - 盈利次数
   - 胜率
   - 总盈亏
3. 帮助您识别最有效的交易方法

### 查看方法盈亏对比

1. 底部柱状图显示 **"各方法盈亏对比"**
2. 每个柱子代表一个方法的总盈亏
3. 直观对比不同方法的表现

## 💡 使用技巧

### 1. Method 库管理技巧

- **分类清晰**：使用统一的分类标准
- **规则详细**：写清楚每一步操作
- **定期回顾**：根据实战效果更新规则
- **删除无效**：淘汰长期表现不佳的方法

### 2. 交易记录技巧

- **及时记录**：交易结束后立即记录
- **详细笔记**：记录市场环境、心态、执行情况
- **标签系统**：使用一致的标签便于筛选
- **诚实记录**：不美化数据，真实反映情况

### 3. 数据分析技巧

- **定期查看**：每周查看一次统计数据
- **对比分析**：对比不同方法、不同货币对的表现
- **趋势观察**：关注盈亏曲线的趋势
- **找出问题**：识别亏损的原因
- **优化策略**：根据数据调整交易计划

## 🔍 数据解读示例

### 示例 1：方法效果分析

**观察数据：**
- 突破交易法：使用 3 次，胜率 100%，盈利 +180
- 回调入场法：使用 2 次，胜率 50%，盈利 +100
- 区间震荡法：使用 2 次，胜率 50%，盈利 -20

**结论：**
- 突破交易法表现最好，应该多用
- 区间震荡法表现不佳，需要优化或暂停使用
- 回调入场法表现中等，需要提高胜率

### 示例 2：货币对选择

**观察数据：**
- EUR/USD：2 笔交易，胜率 100%，盈利 +110
- GBP/USD：1 笔交易，胜率 0%，亏损 -30
- USD/JPY：1 笔交易，胜率 100%，盈利 +70

**结论：**
- EUR/USD 和 USD/JPY 表现好，可以重点关注
- GBP/USD 需要更多数据验证
- 可能需要研究不同货币对的特性

### 示例 3：整体表现评估

**观察数据：**
- 总交易：8 笔
- 胜率：62.5%
- 盈亏因子：7.6
- 平均盈利：76
- 平均亏损：25

**结论：**
- 胜率超过 50%，表现良好
- 盈亏因子很高（>2 就很好），说明盈利交易的收益远大于亏损
- 平均盈利是平均亏损的 3 倍，风险回报比优秀
- 整体交易系统健康，继续保持

## 🎓 学习路径

### 新手阶段（第 1-2 周）
1. 熟悉系统界面和操作
2. 创建 2-3 个基础交易方法
3. 记录每一笔交易
4. 每周查看一次统计

### 进阶阶段（第 3-8 周）
1. 积累 20-50 笔交易数据
2. 分析不同方法的表现
3. 优化交易规则
4. 淘汰低效方法

### 高级阶段（第 9 周以后）
1. 建立稳定的交易系统
2. 持续追踪关键指标
3. 微调和优化策略
4. 保持交易纪律

## 🚨 注意事项

### 当前版本限制
1. **数据不持久化**：重启服务器后数据重置
2. **无备份功能**：暂时无法导出数据
3. **单机使用**：不支持多设备同步

### 临时解决方案
1. 定期截图保存重要数据
2. 手动记录关键统计指标
3. 等待数据库版本更新

### 未来版本
- SQLite 数据库持久化
- 数据导入/导出功能
- 云同步（可选）
