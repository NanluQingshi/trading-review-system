# 交易复盘统计系统

## 🎯 项目概述

这是一个基于 **PriceActionHub** 网站功能的本地化部署版本，专为个人交易者设计的复盘和统计工具。

### 与 PriceActionHub 的对比

| 功能      | PriceActionHub | 本系统     |
| --------- | -------------- | ---------- |
| Method 库 | ✅             | ✅         |
| 交易复盘  | ✅             | ✅         |
| 我的统计  | ✅             | ✅         |
| 部署方式  | 在线平台       | 本地部署   |
| 数据隐私  | 云端存储       | 本地存储   |
| 定制化    | 有限           | 完全可定制 |

## ✨ 核心特性

### 1. Method 库管理 📚

管理您的交易方法库，包括：

- ✅ 创建、编辑、删除交易方法
- ✅ 按分类组织方法（趋势跟踪、震荡交易、形态交易等）
- ✅ 记录详细的交易规则
- ✅ 追踪每个方法的胜率

**示例方法：**

- 突破交易法
- 回调入场法
- 区间震荡法
- Pin Bar 反转

### 2. 交易复盘 📊

详细记录每笔交易：

- ✅ 基本信息：货币对、方向、价格、手数
- ✅ 时间记录：入场时间、出场时间
- ✅ 方法关联：使用的交易方法
- ✅ 结果分析：盈亏、风险回报比
- ✅ 交易笔记：心得和反思
- ✅ 标签系统：灵活分类

**支持的货币对：**

- 主要货币对：EUR/USD, GBP/USD, USD/JPY
- 交叉货币对：EUR/GBP, GBP/JPY, AUD/USD
- 可自定义添加任何交易品种

### 3. 我的统计 📈

多维度数据分析：

**核心指标：**

- 总交易次数
- 胜率百分比
- 总盈亏金额
- 盈亏因子
- 平均盈利/亏损

**可视化图表：**

- 📈 累计盈亏曲线
- 🥧 交易结果分布（盈利/亏损/保本）
- 📊 各方法盈亏对比

**详细统计：**

- 按货币对统计
- 按交易方法统计
- 每日盈亏追踪

## 📦 安装和使用

### 系统要求

- Node.js 14.x 或更高版本
- npm 6.x 或更高版本

### 安装步骤

```bash
# 1. 进入项目目录
cd trading-review-system

# 2. 安装所有依赖
npm run install-all

# 3. 启动应用
npm run dev
```

### 或使用快速启动脚本

```bash
./start.sh
```

### 访问应用

- 前端界面: http://localhost:3000
- 后端 API: http://localhost:5050/api

## 📊 数据模型

### Method（交易方法）

```typescript
{
  id: number;
  name: string; // 方法名称
  description: string; // 简要描述
  category: string; // 分类
  rules: string; // 交易规则
  winRate: number; // 胜率
  createdAt: string; // 创建时间
  updatedAt: string; // 更新时间
}
```

### Trade（交易记录）

```typescript
{
  id: number;
  symbol: string;         // 货币对
  direction: 'long' | 'short';  // 方向
  entryPrice: number;     // 入场价
  exitPrice: number;      // 出场价
  entryTime: string;      // 入场时间
  exitTime: string;       // 出场时间
  lots: number;           // 手数
  profit: number;         // 盈亏
  profitPercent: number;  // 盈亏百分比
  methodId: number;       // 方法ID
  methodName: string;     // 方法名称
  notes: string;          // 交易笔记
  tags: string[];         // 标签
  result: 'win' | 'loss' | 'breakeven';  // 结果
  riskRewardRatio: number;  // 风险回报比
}
```

## 📝 使用建议

### 最佳实践

1. **建立方法库**

   - 先创建 3-5 个核心交易方法
   - 详细记录每个方法的规则
   - 定期回顾和优化

2. **记录交易**

   - 每次交易后立即记录
   - 详细填写交易笔记
   - 诚实记录交易心态

3. **数据分析**

   - 每周查看统计数据
   - 关注胜率和盈亏因子
   - 找出表现最好的方法
   - 识别需要改进的地方

4. **持续改进**
   - 根据数据调整策略
   - 淘汰低效方法
   - 优化高效方法
   - 保持交易纪律
